# KiPy Schematic API Reference v2
# Python API for controlling KiCad schematics via IPC

################################################################################
#                                                                              #
#                    MANDATORY WORKFLOW - READ FIRST!                          #
#                                                                              #
################################################################################

ALWAYS follow this 5-phase workflow when creating schematics. Do NOT skip phases!

PHASE 1: PLANNING (Before ANY tool calls!)
  - List all required components with their library names
  - Draw ASCII topology showing how components connect
  - This phase is THINKING ONLY - no code execution

PHASE 2: PLACEMENT
  - Place ALL symbols first using the standard grid
  - Do NOT add any wires yet
  - Verify all components placed successfully

PHASE 3: PIN DISCOVERY
  - Query actual pin positions from ALL placed symbols
  - Store these exact coordinates - you will need them for wiring
  - Pin positions may not be where you expect!

PHASE 4: WIRING
  - Wire ONLY using exact pin positions from Phase 3
  - Never estimate or round coordinates
  - Add junctions where 3+ wires meet
  - Verify connectivity with sch.get_nets()

PHASE 5: FINISHING
  - Add labels and annotations
  - Run sch.annotate() to assign reference designators
  - Verify no "unconnected" nets exist

================================================================================
HOW TO USE THE SHELL
================================================================================

Use the run_shell tool with mode="sch" to execute Python code for schematic control.

The shell has the following PRE-IMPORTED and READY TO USE:
  - kipy             (the kipy library)
  - sch              (Schematic object - already connected to KiCad!)
  - Vector2          (from kipy.geometry)

IMPORTANT: You do NOT need to import anything - just use `sch` directly!

================================================================================
CRITICAL CODE FORMATTING RULES
================================================================================

1. DO NOT use backslash-n (\n) inside string literals - it causes SyntaxError!
   BAD:  print(f"\nHello")
   GOOD: print("")
         print("Hello")

2. Use 4 spaces for indentation (Python standard)

3. Keep code blocks SHORT - complex operations may timeout

================================================================================
SCHEMATIC PAGE AND GRID - CRITICAL INFORMATION
================================================================================

PAGE SIZE vs USABLE DRAWING AREA:
  - sch.get_page_settings() returns FULL page size (e.g., 297x210mm for A4)
  - The USABLE drawing area is SMALLER due to title block borders!
  - DO NOT use the full page dimensions for layout calculations

USABLE DRAWING AREA (for A4 landscape):
  - X: 20mm to 260mm  (NOT 0 to 297!)
  - Y: 20mm to 180mm  (NOT 0 to 210!)
  - Safe center region: (100-180mm, 60-140mm)

KICAD DEFAULT GRID:
  - Default grid is 50 mils = 1.27mm
  - Components and wires snap to this grid
  - Use positions that align to 1.27mm increments for clean schematics
  - Common grid-aligned positions: 25.4, 26.67, 27.94, 29.21, 30.48... mm

RECOMMENDED COMPONENT SPACING:
  - Minimum horizontal spacing: 25mm (about 20 grid units)
  - Minimum vertical spacing: 15mm (about 12 grid units)
  - Power symbols: place 10-15mm above/below components

STANDARD LAYOUT TEMPLATE:
  Start at (80, 80) and use 25mm horizontal / 20mm vertical spacing:

  Position Grid:
    (80, 80)   (105, 80)   (130, 80)   (155, 80)   (180, 80)
    (80, 100)  (105, 100)  (130, 100)  (155, 100)  (180, 100)
    (80, 120)  (105, 120)  (130, 120)  (155, 120)  (180, 120)

================================================================================
CRITICAL: PIN POSITION BEHAVIOR
================================================================================

WARNING: Pin positions returned by the API may not be where you visually expect!

KNOWN BEHAVIORS:
  - Pin positions may be at or near the symbol CENTER
  - Symbol rotation (angle parameter) may NOT move pins as expected
  - Two pins on a 2-pin component may report nearly identical positions

MANDATORY APPROACH:
  1. Place the symbol
  2. Query pin positions: pos = symbol.pins[n].position
  3. Wire EXACTLY to those coordinates (even if they seem wrong)
  4. KiCad will connect wires that touch the reported pin position

NEVER DO THIS:
  - Assume pins are at specific offsets from symbol center
  - Calculate pin positions based on symbol size or rotation
  - Round or adjust the pin positions returned by the API

EXAMPLE - CORRECT APPROACH:
  # Place symbol
  resistor = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 80))

  # Get ACTUAL positions (don't assume!)
  pin1 = resistor.pins[0].position  # Use this exact value
  pin2 = resistor.pins[1].position  # Use this exact value

  # Wire to exact positions
  sch.add_wire(some_other_pin, pin1)

================================================================================
CRITICAL: WIRE CONNECTION PRECISION
================================================================================

Wires MUST connect to EXACT pin coordinates. Even 0.01mm offset = NO CONNECTION!

PROBLEM EXAMPLE:
  Pin is at:     (195.0000, 104.9619) mm
  Wire goes to:  (195.0000, 105.0000) mm
  Result: "unconnected-(C1-Pad1)" - THE WIRE DOES NOT CONNECT!

SOLUTION - Always use exact pin positions:
  # CORRECT - use the pin position directly
  pin_pos = capacitor.pins[0].position
  sch.add_wire(junction_pos, pin_pos)

  # WRONG - using rounded/estimated position
  sch.add_wire(junction_pos, Vector2.from_xy_mm(195, 105))  # MAY NOT CONNECT!

AFTER WIRING - Always verify connectivity:
  nets = sch.get_nets()
  for net in nets.nets:
      if "unconnected" in net.name:
          print(f"ERROR: {net.name}")  # Must fix these!

================================================================================
IMPORTANT UNITS
================================================================================

KiCad uses NANOMETERS internally for all positions and sizes:
  - 1 mm = 1,000,000 nm
  - Use Vector2.from_xy_mm(x_mm, y_mm) for millimeter input (RECOMMENDED)
  - Use Vector2.from_xy(x_nm, y_nm) for nanometer input

Converting from API positions to mm for debugging:
  x_mm = pin.position.x / 1e6
  y_mm = pin.position.y / 1e6

class Vector2:
    """2D vector/point in nanometers"""

    # Creation methods:
    Vector2.from_xy(x_nm: int, y_nm: int) -> Vector2       # From nanometers
    Vector2.from_xy_mm(x_mm: float, y_mm: float) -> Vector2  # From millimeters (RECOMMENDED)

    # Properties:
    .x -> int                  # X coordinate in nanometers
    .y -> int                  # Y coordinate in nanometers

    # Methods:
    .length() -> float         # Vector magnitude
    .angle() -> float          # Angle in radians
    .angle_degrees() -> float  # Angle in degrees

    # Operators: +, -, *, negation supported

class Symbol:
    """A component/symbol in the schematic"""
    .id -> KIID                # Unique identifier
    .lib_id -> LibraryIdentifier
    .position -> Vector2
    .unit -> int               # Unit number (for multi-unit)
    .angle -> float            # Rotation in degrees (may not affect pins!)
    .mirror_x -> bool
    .mirror_y -> bool
    .dnp -> bool               # Do Not Populate flag
    .fields -> List[Field]     # Reference, Value, Footprint, etc.
    .pins -> List[Pin]         # Symbol pins - ALWAYS query these for wiring!

    .get_field(name: str) -> Field | None

class Pin:
    """A pin on a symbol - ALWAYS use .position for wire connections!"""
    .name -> str               # Pin name
    .number -> str             # Pin number
    .position -> Vector2       # ACTUAL position - use this for wiring!
    .angle -> float            # Pin orientation

class Field:
    """A text field on a symbol"""
    .name -> str               # Field name (Reference, Value, Footprint, etc.)
    .text -> str               # Field value
    .position -> Vector2

class Wire:
    """An electrical wire"""
    .id -> KIID
    .start -> Vector2
    .end -> Vector2
    .layer -> int              # 0=Wire, 2=Bus

class Junction:
    """A junction/connection point"""
    .id -> KIID
    .position -> Vector2

class NoConnect:
    """A no-connect marker"""
    .id -> KIID
    .position -> Vector2

class LocalLabel:
    """A local net label"""
    .id -> KIID
    .position -> Vector2

class GlobalLabel:
    """A global net label"""
    .id -> KIID
    .position -> Vector2

class Sheet:
    """A hierarchical sheet reference"""
    .id -> KIID
    .position -> Vector2
    .size -> Vector2
    .name -> str
    .filename -> str
    .pins -> List[SheetPin]

class SheetPin:
    """A pin on a hierarchical sheet"""
    .id -> KIID
    .name -> str
    .position -> Vector2
    .side -> int               # 1=Left, 2=Right, 3=Top, 4=Bottom
    .shape -> int              # 1=Input, 2=Output, 3=Bidirectional

================================================================================
COMMON LIBRARY SYMBOLS
================================================================================

Passive Components (use sch.add_symbol with "Library:Symbol" format):
  "Device:R"              - Resistor (2 pins)
  "Device:C"              - Capacitor (2 pins)
  "Device:L"              - Inductor (2 pins)
  "Device:LED"            - LED (2 pins: A, K)
  "Device:D"              - Diode (2 pins: A, K)
  "Device:D_Schottky"     - Schottky diode (2 pins: A, K)
  "Device:D_Zener"        - Zener diode (2 pins: A, K)

Transistors (NOTE: Pin order varies! Always query pin positions!):
  "Transistor_FET:BSS138"           - N-channel MOSFET (G, S, D)
  "Transistor_FET:IRF540N"          - Power N-channel MOSFET
  "Transistor_BJT:2N2222"           - NPN transistor (B, C, E)
  "Transistor_BJT:2N3904"           - NPN transistor (B, C, E)
  WARNING: Generic symbols like "Device:Q_NMOS_GDS" may NOT exist!
           Use specific part numbers from Transistor_FET or Transistor_BJT libraries.

Power Symbols (use sch.add_power_symbol with JUST the name, no library prefix):
  "GND"               - Ground
  "VCC"               - VCC power rail
  "+3V3"              - 3.3V rail
  "+5V"               - 5V rail
  "+12V"              - 12V rail
  "VDD"               - Digital supply
  "VSS"               - Digital ground

Connectors:
  "Connector:Conn_01x02_Pin"  - 2-pin header
  "Connector:Conn_01x04_Pin"  - 4-pin header
  "Connector:Conn_01x06_Pin"  - 6-pin header

================================================================================
COMPLETE WORKFLOW EXAMPLE - VOLTAGE DIVIDER
================================================================================

# ============================================================
# PHASE 1: PLANNING (do this mentally before any code!)
# ============================================================
# Components needed:
#   - R1: Device:R (top resistor)
#   - R2: Device:R (bottom resistor)
#   - VCC: power symbol (input)
#   - GND: power symbol (ground)
#
# Topology:
#   VCC
#    |
#   R1
#    +---- VOUT
#   R2
#    |
#   GND
#
# Positions (25mm spacing):
#   VCC at (100, 60)
#   R1  at (100, 80)
#   R2  at (100, 110)
#   GND at (100, 130)

# ============================================================
# PHASE 2: PLACEMENT - Place all symbols, no wires yet
# ============================================================
vcc = sch.add_power_symbol("VCC", Vector2.from_xy_mm(100, 60))
r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 80))
r2 = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 110))
gnd = sch.add_power_symbol("GND", Vector2.from_xy_mm(100, 130))
print("Phase 2 complete: All symbols placed")

# ============================================================
# PHASE 3: PIN DISCOVERY - Get exact pin positions
# ============================================================
vcc_pin = vcc.pins[0].position
r1_pin1 = r1.pins[0].position
r1_pin2 = r1.pins[1].position
r2_pin1 = r2.pins[0].position
r2_pin2 = r2.pins[1].position
gnd_pin = gnd.pins[0].position

# Debug: print positions to verify
print(f"VCC pin: ({vcc_pin.x/1e6:.2f}, {vcc_pin.y/1e6:.2f})")
print(f"R1 pins: ({r1_pin1.x/1e6:.2f}, {r1_pin1.y/1e6:.2f}), ({r1_pin2.x/1e6:.2f}, {r1_pin2.y/1e6:.2f})")
print(f"R2 pins: ({r2_pin1.x/1e6:.2f}, {r2_pin1.y/1e6:.2f}), ({r2_pin2.x/1e6:.2f}, {r2_pin2.y/1e6:.2f})")
print(f"GND pin: ({gnd_pin.x/1e6:.2f}, {gnd_pin.y/1e6:.2f})")

# ============================================================
# PHASE 4: WIRING - Use exact pin positions only!
# ============================================================
sch.add_wire(vcc_pin, r1_pin1)      # VCC to R1 top
sch.add_wire(r1_pin2, r2_pin1)      # R1 bottom to R2 top (junction point)
sch.add_wire(r2_pin2, gnd_pin)      # R2 bottom to GND

# Add junction at voltage divider output
junction_pos = r1_pin2  # Use exact position!
sch.add_junction(junction_pos)

# Add output wire from junction
sch.add_wire(junction_pos, Vector2.from_xy_mm(130, junction_pos.y/1e6))

# Verify connectivity
nets = sch.get_nets()
print("")
print("Nets:")
for net in nets.nets:
    status = "OK" if "unconnected" not in net.name else "ERROR!"
    print(f"  {net.name} - {status}")

# ============================================================
# PHASE 5: FINISHING - Labels and annotation
# ============================================================
sch.add_label("VOUT", Vector2.from_xy_mm(125, junction_pos.y/1e6 - 3))
sch.annotate(scope="all", start_number=1)
print("")
print("Schematic complete!")

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "unconnected-(XX-PadN)" in nets
CAUSE: Wire endpoint doesn't exactly match pin position
FIX: Use exact pin.position values, don't round or estimate

PROBLEM: Symbol rotation doesn't change pin positions
CAUSE: Known API limitation - rotation may not transform pins
FIX: Always query pin positions after placement, don't calculate them

PROBLEM: Components placed outside visible area
CAUSE: Using full page dimensions instead of usable area
FIX: Stay within (20-260mm, 20-180mm) for A4, or (80-180mm, 60-140mm) to be safe

PROBLEM: global labels cause API error
CAUSE: Global label creation may not be fully supported
FIX: Use label_type="local" instead

PROBLEM: sch.save() or sch.refresh() fails
CAUSE: These commands may not be implemented in the API
FIX: User must save manually in KiCad (Ctrl+S)
