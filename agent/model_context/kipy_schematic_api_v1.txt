# KiPy Schematic API Reference
# Python API for controlling KiCad schematics via IPC

================================================================================
HOW TO USE THE SHELL
================================================================================

Use the run_shell tool with mode="sch" to execute Python code for schematic control.

The shell has the following PRE-IMPORTED and READY TO USE:
  - kipy             (the kipy library)
  - sch              (Schematic object - already connected to KiCad!)
  - Vector2          (from kipy.geometry)

IMPORTANT: You do NOT need to import anything - just use `sch` directly!

================================================================================
CRITICAL CODE FORMATTING RULES
================================================================================

1. DO NOT use backslash-n (\n) inside string literals - it causes SyntaxError!
   BAD:  print(f"\nHello")
   GOOD: print("")
         print("Hello")

2. Use 4 spaces for indentation (Python standard)

3. Keep code blocks SHORT - complex operations may timeout

================================================================================
SCHEMATIC DESIGN WORKFLOW (FOLLOW THIS ORDER!)
================================================================================

For reliable schematic creation, follow this 4-phase workflow:

PHASE 1: PART PLACEMENT
  - Choose all components needed for the circuit
  - Place each component at well-spaced positions (20-30mm apart minimum)
  - Use positions in the visible area (100-200mm X, 80-150mm Y)
  - Print pin positions for EACH component after placing it

PHASE 2: CONNECTION PLANNING (do this mentally or in comments)
  - Define what connects to what conceptually:
    "VCC pin connects to R1 pin 1"
    "R1 pin 2 connects to Q1 gate"
  - Identify switch nodes and power rails
  - Plan wire routing (what goes horizontal vs vertical)

PHASE 3: WIRING (using EXACT pin positions)
  - Retrieve actual pin positions from each symbol
  - Wire using ONLY the positions returned by the API
  - NEVER use rounded or "cleaned up" positions
  - Store pin positions in variables before wiring

PHASE 4: VALIDATION
  - List all wires and verify they connect to intended pins
  - Check that wire endpoints match pin positions exactly
  - Add junctions where 3+ wires meet
  - Run ERC if available

================================================================================
SCHEMATIC PAGE COORDINATES
================================================================================

KiCad schematic pages are typically A4 (297x210mm) or Letter size.
The visible drawing area starts at approximately:
  - X: 10mm to 280mm
  - Y: 10mm to 190mm

RECOMMENDED: Place components in the range (100-200mm, 80-150mm) to ensure
they appear in the visible page area.

Quick Start Examples:
  # List all symbols in the schematic
  print(sch.get_symbols())

  # Add a resistor at a VISIBLE position (center of typical page)
  r = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 100))

  # Get page size to calculate positions
  page = sch.get_page_settings()
  center_x = page.width_mm / 2
  center_y = page.height_mm / 2
  print(f"Page center: {center_x}, {center_y}")

================================================================================
IMPORTANT UNITS
================================================================================

KiCad uses NANOMETERS internally for all positions and sizes:
  - 1 mm = 1,000,000 nm
  - Use Vector2.from_xy_mm(x_mm, y_mm) for millimeter input (RECOMMENDED)
  - Use Vector2.from_xy(x_nm, y_nm) for nanometer input

================================================================================
CRITICAL: PIN POSITIONS ARE CORRECT - USE THEM EXACTLY!
================================================================================

When you read pin positions from placed symbols, they may appear to have very
small offsets from the symbol center. THIS IS CORRECT BEHAVIOR - do NOT try
to "fix" or "round" these positions!

Example output you might see:
  MOSFET pins:
    D(2): (110.025, 79.949) mm
    G(1): (109.949, 80.000) mm
    S(3): (110.025, 80.051) mm

CORRECT behavior: Use these EXACT positions for wiring!
  drain_pos = mosfet.pins[0].position  # Use directly!
  sch.add_wire(some_position, drain_pos)

WRONG behavior (DO NOT DO THIS):
  # NEVER round or "clean up" positions like this:
  sch.add_wire(Vector2.from_xy_mm(110, 80), ...)  # BAD! Won't connect!

The small offsets represent the actual pin locations on the symbol graphics.
Wires MUST connect to these exact positions to make electrical connections.

RULE: Always store pin.position in a variable and use it directly for wiring.
      Never substitute a "cleaner" coordinate - it will NOT connect!

class Vector2:
    """2D vector/point in nanometers"""

    # Creation methods:
    Vector2.from_xy(x_nm: int, y_nm: int) -> Vector2       # From nanometers
    Vector2.from_xy_mm(x_mm: float, y_mm: float) -> Vector2  # From millimeters (RECOMMENDED)

    # Properties:
    .x -> int                  # X coordinate in nanometers
    .y -> int                  # Y coordinate in nanometers

    # Methods:
    .length() -> float         # Vector magnitude
    .angle() -> float          # Angle in radians
    .angle_degrees() -> float  # Angle in degrees

    # Operators: +, -, *, negation supported

SCHEMATIC TYPES

class Symbol:
    """A component/symbol in the schematic"""
    .id -> KIID                # Unique identifier
    .lib_id -> LibraryIdentifier
    .position -> Vector2
    .unit -> int               # Unit number (for multi-unit)
    .angle -> float            # Rotation in degrees
    .mirror_x -> bool
    .mirror_y -> bool
    .dnp -> bool               # Do Not Populate flag
    .fields -> List[Field]     # Reference, Value, Footprint, etc.
    .pins -> List[Pin]         # Symbol pins WITH ACTUAL SCHEMATIC POSITIONS

    .get_field(name: str) -> Field | None

class Pin:
    """A pin on a symbol - use .position for wire connections!"""
    .name -> str               # Pin name
    .number -> str             # Pin number
    .position -> Vector2       # ACTUAL position in schematic coordinates
    .angle -> float            # Pin orientation

class Field:
    """A text field on a symbol"""
    .name -> str               # Field name (Reference, Value, Footprint, etc.)
    .text -> str               # Field value
    .position -> Vector2

class Wire:
    """An electrical wire"""
    .id -> KIID
    .start -> Vector2
    .end -> Vector2
    .layer -> int              # 0=Wire, 2=Bus

class Junction:
    """A junction/connection point"""
    .id -> KIID
    .position -> Vector2

class NoConnect:
    """A no-connect marker"""
    .id -> KIID
    .position -> Vector2

class LocalLabel:
    """A local net label"""
    .id -> KIID
    .position -> Vector2

class GlobalLabel:
    """A global net label"""
    .id -> KIID
    .position -> Vector2

class Sheet:
    """A hierarchical sheet reference"""
    .id -> KIID
    .position -> Vector2
    .size -> Vector2
    .name -> str
    .filename -> str
    .pins -> List[SheetPin]

class SheetPin:
    """A pin on a hierarchical sheet"""
    .id -> KIID
    .name -> str
    .position -> Vector2
    .side -> int               # 1=Left, 2=Right, 3=Top, 4=Bottom
    .shape -> int              # 1=Input, 2=Output, 3=Bidirectional

================================================================================
USAGE EXAMPLES
================================================================================

# Example 1: Place a resistor and connect with wires using ACTUAL PIN POSITIONS
# CRITICAL: Always use symbol.pins[n].position for wire connections!
r = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 100), angle=90.0)

# Get ACTUAL pin positions from the placed symbol
pin1_pos = r.pins[0].position
pin2_pos = r.pins[1].position

# Connect wires TO the actual pin positions
sch.add_wire(Vector2.from_xy_mm(120, 100), pin1_pos)
sch.add_wire(pin2_pos, Vector2.from_xy_mm(180, 100))

# Example 2: Create a simple circuit - ALWAYS use .pins[n].position for wires!
vcc = sch.add_power_symbol("VCC", Vector2.from_xy_mm(150, 70), angle=180)
gnd = sch.add_power_symbol("GND", Vector2.from_xy_mm(150, 130))
resistor = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 100))

# Wire from VCC pin to resistor pin (use actual positions!)
sch.add_wire(vcc.pins[0].position, resistor.pins[0].position)
sch.add_wire(resistor.pins[1].position, gnd.pins[0].position)

# Example 3: List all components (no \n in strings!)
for sym in sch.get_symbols():
    ref = sym.get_field("Reference")
    val = sym.get_field("Value")
    ref_text = ref.text if ref else "?"
    val_text = val.text if val else "?"
    print(f"{ref_text}: {val_text}")

# Example 4: Annotate (note: save/refresh may not be supported on all servers)
result = sch.annotate(scope="all", start_number=1)
print(f"Annotated {result.symbols_annotated} symbols")

# Example 5: Get pin positions for wiring (IMPORTANT PATTERN)
symbols = sch.get_symbols()
for sym in symbols:
    ref = sym.get_field("Reference")
    name = ref.text if ref else "?"
    print(f"Symbol {name} pins:")
    for pin in sym.pins:
        px = pin.position.x / 1e6  # Convert nm to mm
        py = pin.position.y / 1e6
        print(f"  {pin.name}({pin.number}): ({px:.2f}, {py:.2f}) mm")

COMMON LIBRARY SYMBOLS

Passive Components (use sch.add_symbol with "Library:Symbol" format):
  "Device:R"              - Resistor
  "Device:C"              - Capacitor
  "Device:L"              - Inductor
  "Device:LED"            - LED

Transistors (NOTE: Pin order varies! Check library for exact names):
  "Transistor_FET:BSS138"           - N-channel MOSFET (common, reliable)
  "Transistor_FET:IRF540N"          - Power N-channel MOSFET
  "Transistor_BJT:2N2222"           - NPN transistor
  "Transistor_BJT:2N3904"           - NPN transistor
  WARNING: Generic symbols like "Device:Q_NMOS_GDS" may NOT exist!
           Use specific part numbers from Transistor_FET or Transistor_BJT libraries.

Power Symbols (use sch.add_power_symbol with JUST the name, no library prefix):
  "GND"               - Ground
  "VCC"               - VCC power rail
  "+3V3"              - 3.3V rail
  "+5V"               - 5V rail
  "+12V"              - 12V rail
  "VDD"               - Digital supply
  "VSS"               - Digital ground

Connectors:
  "Connector:Conn_01x02_Pin"  - 2-pin header
  "Connector:Conn_01x04_Pin"  - 4-pin header
  "Connector:Conn_01x06_Pin"  - 6-pin header

================================================================================
WIRING BEST PRACTICES
================================================================================

CRITICAL RULES:
1. ALWAYS place all symbols FIRST, THEN add wires
2. ALWAYS use pin.position directly - NEVER use rounded coordinates
3. Store pin positions in variables BEFORE creating wires
4. Use junctions where 3+ wires meet

CORRECT wiring pattern (components placed at different Y positions):
  # Place components
  r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 100))
  c1 = sch.add_symbol("Device:C", Vector2.from_xy_mm(180, 120))

  # Store EXACT pin positions (do NOT modify these!)
  r1_pin2 = r1.pins[1].position
  c1_pin1 = c1.pins[0].position

  # Wire with L-shape routing (horizontal then vertical)
  # Create intermediate point at same X as c1, same Y as r1_pin2
  mid_x = c1_pin1.x
  mid_y = r1_pin2.y
  mid_point = Vector2.from_xy(mid_x, mid_y)

  sch.add_wire(r1_pin2, mid_point)       # Horizontal segment
  sch.add_wire(mid_point, c1_pin1)       # Vertical segment

CORRECT wiring for aligned components:
  # When pins are already aligned (same X or same Y)
  r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 100))
  r2 = sch.add_symbol("Device:R", Vector2.from_xy_mm(150, 130))  # Same X

  # Direct vertical wire using EXACT positions
  sch.add_wire(r1.pins[1].position, r2.pins[0].position)

WRONG (DO NOT DO THIS):
  # NEVER use hardcoded positions when pin positions are available!
  sch.add_wire(Vector2.from_xy_mm(150, 100), Vector2.from_xy_mm(180, 100))  # BAD!
