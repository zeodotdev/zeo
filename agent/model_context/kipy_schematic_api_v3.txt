# KiPy Schematic API Reference v3
# Python API for controlling KiCad schematics via IPC

################################################################################
#                                                                              #
#                    MANDATORY WORKFLOW - READ FIRST!                          #
#                                                                              #
################################################################################

ALWAYS follow this 5-phase workflow when creating schematics. Do NOT skip phases!

PHASE 1: PLANNING (Before ANY tool calls!)
  - Query page settings and usable area: sch.get_usable_area()
  - Query grid settings: sch.get_grid_settings()
  - Note the CENTER position (typically 140, 100 for A4)
  - List all required components with their library names
  - Draw ASCII topology showing how components connect
  - Plan positions CENTERED in usable area, aligned to grid
  - Keep total design within ~100mm x 80mm footprint

PHASE 2: PLACEMENT
  - *** START AT CENTER: area['center_x_mm'], area['center_y_mm'] ***
  - Use sch.snap_to_grid(x, y) to align positions to grid
  - Place ALL symbols first using grid-aligned positions
  - Use 30-40mm spacing between components (not 20-25mm)
  - Do NOT add any wires yet
  - Verify all components placed successfully

PHASE 3: PIN DISCOVERY
  - Query pin positions using sch.get_pin_position(symbol, pin_id)
  - Print pin positions to verify they're correct
  - Note: Pin positions are EXACT - use them without modification

PHASE 4: WIRING (Use Pin-Based Wiring!)
  - ALWAYS use pin-based wiring functions:
    * sch.wire_pins(sym1, pin1, sym2, pin2) - Direct pin-to-pin
    * sch.wire_path((sym1, pin1), waypoints, (sym2, pin2)) - Routed path
    * sch.wire_from_pin(sym, pin, (x, y)) - Pin to coordinate
    * sch.wire_to_pin((x, y), sym, pin) - Coordinate to pin
  - Add junctions where 3+ wires meet
  - Check for unconnected pins: sch.get_unconnected_pins()

PHASE 5: FINISHING
  - Add labels and annotations
  - Run sch.annotate() to assign reference designators
  - Verify no unconnected pins: sch.get_unconnected_pins()

================================================================================
CRITICAL CODE FORMATTING RULES
================================================================================

1. DO NOT use backslash-n (\n) inside string literals - it causes SyntaxError!
   BAD:  print(f"\nHello")
   GOOD: print("")
         print("Hello")

2. Use 4 spaces for indentation (Python standard)

3. Keep code blocks SHORT - complex operations may timeout

================================================================================
SCHEMATIC PAGE AND COORDINATE SYSTEM - CRITICAL!
================================================================================

*** COORDINATE SYSTEM (MUST UNDERSTAND!) ***
  - Page origin (0, 0) is at the TOP-LEFT corner of the page
  - X increases to the RIGHT (towards right edge)
  - Y increases DOWNWARD (towards bottom edge)
  - ALL COORDINATES are in MILLIMETERS when using Vector2.from_xy_mm()
  - Internal API uses nanometers (1 mm = 1,000,000 nm)

ALWAYS START by querying the usable area and grid:

  # Get usable drawing area (accounts for title block margins)
  area = sch.get_usable_area()
  print(f"Usable area: X={area['min_x_mm']:.0f}-{area['max_x_mm']:.0f}mm")
  print(f"            Y={area['min_y_mm']:.0f}-{area['max_y_mm']:.0f}mm")
  print(f"Center: ({area['center_x_mm']:.0f}, {area['center_y_mm']:.0f})")
  print(f"Full page: {area['page_width_mm']:.0f}x{area['page_height_mm']:.0f}mm")

  # Get grid settings (NOTE: returns defaults - no API to query actual grid!)
  grid = sch.get_grid_settings()
  print(f"Grid: {grid['size_mils']} mils = {grid['size_mm']}mm")
  if grid['is_default']:
      print("(Using default grid - actual grid in KiCad may differ)")

TYPICAL VALUES (A4 landscape, 297x210mm page):
  - Usable X: 10mm to 262mm (left margin to title block)
  - Usable Y: 10mm to 185mm (top margin to title block)
  - Center: (136, 97) approximately
  - Grid: 50 mils = 1.27mm (DEFAULT VALUE)

*** CRITICAL: CENTER YOUR DESIGN! ***
  - ALWAYS start placement at the CENTER of the usable area
  - For A4: Start at (~136, ~97), NOT at (60, 60)!
  - Spread components outward from center
  - Use LARGE spacing: 30-50mm between components (not 20-25mm)

ALWAYS snap positions to grid:
  x, y = sch.snap_to_grid(100.5, 75.3)  # Returns (100.33, 76.2) or similar
  pos = Vector2.from_xy_mm(x, y)  # Creates position in MILLIMETERS

GRID AND COORDINATE HELPERS (Phase 1 API)

sch.get_grid_settings() -> dict
    """Get the grid settings (returns DEFAULTS - no API to query actual grid).

    IMPORTANT: KiCad IPC API does not expose grid settings. This function
    returns the default KiCad schematic grid values. If you've changed the
    grid in KiCad preferences, these values may not match.

    Returns:
        dict with:
        - size_mm: Grid size in millimeters (default: 1.27mm)
        - size_mils: Grid size in mils (default: 50)
        - size_nm: Grid size in nanometers
        - is_default: True (indicates these are default values)

    Example:
        grid = sch.get_grid_settings()
        print(f"Grid: {grid['size_mils']} mils = {grid['size_mm']}mm")
        if grid['is_default']:
            print("Note: Using default grid values")
    """

sch.snap_to_grid(x_mm: float, y_mm: float, grid_mm: float = 1.27) -> Tuple[float, float]
    """Snap coordinates to nearest grid point.

    Args:
        x_mm: X coordinate in MILLIMETERS
        y_mm: Y coordinate in MILLIMETERS
        grid_mm: Grid size in millimeters (default: 1.27mm = 50 mils)

    Returns:
        Tuple of (x_mm, y_mm) snapped to grid (in MILLIMETERS)

    Example:
        x, y = sch.snap_to_grid(100.5, 75.3)  # Returns grid-aligned coords
        pos = Vector2.from_xy_mm(x, y)  # Create Vector2 from MM values
        symbol = sch.add_symbol("Device:R", pos)
    """

sch.get_usable_area() -> dict
    """Get usable drawing area (accounting for title block margins).

    Returns:
        dict with (all values in MILLIMETERS):
        - min_x_mm, max_x_mm: Horizontal bounds (left to right)
        - min_y_mm, max_y_mm: Vertical bounds (top to bottom)
        - left_mm, right_mm, top_mm, bottom_mm: Alternative names
        - width_mm, height_mm: Usable dimensions
        - center_x_mm, center_y_mm: Center point for placement
        - page_width_mm, page_height_mm: Full page dimensions

    Example:
        area = sch.get_usable_area()
        print(f"Usable: {area['min_x_mm']:.0f}-{area['max_x_mm']:.0f}mm")
        # Place component in CENTER (recommended starting point!)
        pos = Vector2.from_xy_mm(area['center_x_mm'], area['center_y_mm'])
    """

================================================================================
PIN-BASED WIRING (Phase 2 API) - ALWAYS USE THESE!
================================================================================

These functions eliminate wire connection errors by using exact pin positions.

sch.get_pin_position(symbol, pin_id: str) -> Vector2 | None
    """Get exact position of a pin by name or number.

    Args:
        symbol: The symbol containing the pin
        pin_id: Pin name ("VCC", "GND") or number ("1", "2")

    Returns:
        Vector2 position or None if not found

    Example:
        r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 80))
        pin1_pos = sch.get_pin_position(r1, "1")
        pin2_pos = sch.get_pin_position(r1, "2")
    """

sch.wire_pins(symbol1, pin_id1: str, symbol2, pin_id2: str) -> Wire | None
    """Wire directly between two pins. RECOMMENDED!

    This uses exact pin positions - no coordinate errors possible.

    Example:
        r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 80))
        r2 = sch.add_symbol("Device:R", Vector2.from_xy_mm(100, 110))
        wire = sch.wire_pins(r1, "2", r2, "1")  # Connect R1 pin 2 to R2 pin 1
    """

sch.wire_path(start_pin, waypoints, end_pin) -> List[Wire]
    """Wire from pin through waypoints to pin. For L-shaped or complex routes.

    Args:
        start_pin: Tuple of (symbol, pin_id)
        waypoints: List of (x_mm, y_mm) tuples
        end_pin: Tuple of (symbol, pin_id)

    Example:
        # Route R1 pin 2 -> right -> down -> R2 pin 1
        wires = sch.wire_path(
            (r1, "2"),
            [(120, 80), (120, 100)],  # Go right, then down
            (r2, "1")
        )
    """

sch.wire_from_pin(symbol, pin_id: str, end_point_mm: Tuple[float, float]) -> Wire | None
    """Wire from a pin to a coordinate (for junctions).

    Example:
        cap = sch.add_symbol("Device:C", Vector2.from_xy_mm(120, 80))
        wire = sch.wire_from_pin(cap, "1", (100, 80))  # Pin to junction
    """

sch.wire_to_pin(start_point_mm: Tuple[float, float], symbol, pin_id: str) -> Wire | None
    """Wire from a coordinate to a pin.

    Example:
        wire = sch.wire_to_pin((100, 100), cap, "2")  # Junction to pin
    """

sch.get_unconnected_pins() -> List[dict]
    """Get list of unconnected pins for validation.

    Returns:
        List of dicts with: net_name, symbol_ref, pin_number

    Example:
        unconnected = sch.get_unconnected_pins()
        if unconnected:
            print("WARNING: Unconnected pins!")
            for pin in unconnected:
                print(f"  {pin['net_name']}")
        else:
            print("All pins connected!")
    """

================================================================================
AUTO-WIRING WITH L-SHAPED ROUTING (RECOMMENDED FOR MOST CONNECTIONS!)
================================================================================

These functions automatically create clean orthogonal (right-angle) wire routes.
Use these instead of wire_pins() when pins are not aligned!

sch.auto_wire(symbol1, pin_id1, symbol2, pin_id2, style="L") -> List[Wire]
    """Automatically wire between two pins using L-shaped orthogonal routing.

    This creates clean right-angle wire routes instead of diagonal lines.
    The routing automatically chooses horizontal-first or vertical-first
    based on the relative positions of the pins.

    Args:
        symbol1: First symbol
        pin_id1: Pin on first symbol (name or number)
        symbol2: Second symbol
        pin_id2: Pin on second symbol (name or number)
        style: Routing style:
               - "L": Auto-choose best L-route (DEFAULT - use this!)
               - "L_horizontal_first": Go horizontal, then vertical
               - "L_vertical_first": Go vertical, then horizontal
               - "direct": Single straight wire (may be diagonal)

    Returns:
        List of Wire objects (1 wire if aligned, 2 wires for L-route)

    Example:
        mosfet = sch.add_symbol("Device:Q_NMOS_GSD", Vector2.from_xy_mm(100, 80))
        resistor = sch.add_symbol("Device:R", Vector2.from_xy_mm(130, 60))
        # Creates L-shaped route from drain to resistor
        wires = sch.auto_wire(mosfet, "D", resistor, "1")
    """

sch.auto_wire_to_point(symbol, pin_id, point_mm, style="L") -> List[Wire]
    """Automatically wire from a pin to a coordinate point.

    Args:
        symbol: The symbol containing the pin
        pin_id: Pin identifier (name or number)
        point_mm: Target point as (x_mm, y_mm) tuple
        style: Routing style (same as auto_wire)

    Example:
        # Wire from MOSFET source to switch node junction
        wires = sch.auto_wire_to_point(mosfet, "S", (100, 90))
    """

sch.connect_to_junction(pins, junction_mm, style="L") -> Tuple[List[Wire], Junction]
    """Connect multiple pins to a common junction point (star connection).

    This is perfect for nodes like switch nodes in buck converters where
    multiple components meet at one point.

    Args:
        pins: List of (symbol, pin_id) tuples
        junction_mm: Junction location as (x_mm, y_mm)
        style: Routing style for each wire

    Returns:
        Tuple of (list of all wires, junction object)

    Example:
        # Buck converter switch node: MOSFET source + diode cathode + inductor
        wires, junction = sch.connect_to_junction(
            [(mosfet, "S"), (diode, "K"), (inductor, "1")],
            junction_mm=(100, 90)
        )
    """

================================================================================
IMPORTANT: WHY USE PIN-BASED WIRING?
================================================================================

PROBLEM: Manual coordinate wiring is error-prone:
  # This might NOT connect due to tiny position errors!
  sch.add_wire(Vector2.from_xy_mm(100, 80), Vector2.from_xy_mm(100, 110))

Even 0.01mm offset = "unconnected" error!

SOLUTION: Pin-based wiring uses EXACT positions:
  # This ALWAYS connects correctly!
  sch.wire_pins(r1, "2", r2, "1")

ALWAYS prefer (in order of preference):
  1. sch.auto_wire() - L-shaped routing, works for any pin positions
  2. sch.connect_to_junction() - for star connections (3+ components to one point)
  3. sch.wire_pins() - direct connections when pins are aligned
  4. sch.auto_wire_to_point() - pin to coordinate with L-routing
  5. sch.wire_from_pin() / sch.wire_to_pin() - for manual junction wiring

================================================================================
PIN POSITION NOTES
================================================================================

PIN ROTATION SUPPORT:
  The get_pin_position() function now automatically transforms pin positions
  to account for symbol rotation and mirroring. This means pins on rotated
  symbols will report their correct world-space positions.

CLUSTERING NOTE: Some symbols (especially transistors) have pins very close
together in the symbol definition. This is normal - use auto_wire() which
handles this correctly.

RECOMMENDED APPROACH FOR MULTI-PIN COMPONENTS:
  1. Place components using add_symbol() with desired angle/mirror
  2. Use auto_wire() or connect_to_junction() for wiring - these use exact
     pin positions automatically
  3. For complex nodes (3+ components), use connect_to_junction()
  4. Prefer generic symbols (Device:Q_NMOS_GSD) for better pin spacing

================================================================================
IMPORTANT UNITS - READ CAREFULLY!
================================================================================

*** COORDINATE VALUES - AVOID CONFUSION! ***

When placing components or creating wires, ALWAYS use MILLIMETERS:
  - Standard A4 page is ~297mm x 210mm
  - Typical center is around (136, 97) mm
  - Component positions should be in range 10-280mm for X, 10-185mm for Y

WRONG (values too large - these would be meters!):
  pos = Vector2.from_xy_mm(175000, 195000)  # 175 METERS - WRONG!

CORRECT (reasonable millimeter values):
  pos = Vector2.from_xy_mm(136, 97)  # 136mm, 97mm - CORRECT!

KiCad uses NANOMETERS internally:
  - 1 mm = 1,000,000 nm (one million nanometers)
  - Use Vector2.from_xy_mm(x_mm, y_mm) for millimeter input (RECOMMENDED!)
  - Use Vector2.from_xy(x_nm, y_nm) only if you need nanometer precision

Converting from API positions to mm for debugging:
  x_mm = pin.position.x / 1e6  # Divide nm by 1,000,000 to get mm
  y_mm = pin.position.y / 1e6

*** SANITY CHECK FOR COORDINATES ***
  - If X or Y > 500, you're probably using wrong units!
  - A4 page is only 297mm wide - values like 175000 are WRONG
  - Always check: area = sch.get_usable_area() to see valid ranges

class Symbol:
    """A component/symbol in the schematic"""
    .id -> KIID                # Unique identifier
    .lib_id -> LibraryIdentifier
    .position -> Vector2
    .unit -> int               # Unit number (for multi-unit)
    .angle -> float            # Rotation in degrees (may not affect pins!)

    .mirror_y -> bool

    .fields -> List[Field]     # Reference, Value, Footprint, etc.
    .pins -> List[Pin]         # Symbol pins - ALWAYS query these for wiring!

    .get_field(name: str) -> Field | None

class Pin:
    """A pin on a symbol - ALWAYS use .position for wire connections!"""

    .number -> str             # Pin number

    .text -> str               # Field value
    .position -> Vector2

    .id -> KIID
    .start -> Vector2
    .end -> Vector2
    .layer -> int              # 0=Wire, 2=Bus

class Junction:
    """A junction/connection point"""
    .id -> KIID
    .position -> Vector2

class NoConnect:
    """A no-connect marker"""

    .position -> Vector2

class LocalLabel:
    """A local net label"""
    .id -> KIID
    .position -> Vector2

class GlobalLabel:
    """A global net label"""
    .id -> KIID
    .position -> Vector2

class Sheet:

================================================================================
COMMON LIBRARY SYMBOLS
================================================================================

Passive Components (use sch.add_symbol with "Library:Symbol" format):
  "Device:R"              - Resistor (2 pins)
  "Device:C"              - Capacitor (2 pins)
  "Device:L"              - Inductor (2 pins)
  "Device:LED"            - LED (2 pins: A, K)
  "Device:D"              - Diode (2 pins: A, K)
  "Device:D_Schottky"     - Schottky diode (2 pins: A, K)
  "Device:D_Zener"        - Zener diode (2 pins: A, K)

Transistors:
  *** PREFER GENERIC SYMBOLS - they have better pin position reporting! ***

  Generic (RECOMMENDED - pins may be more spread out):
  "Device:Q_NMOS_GSD"             - Generic N-MOSFET (G, S, D pins)
  "Device:Q_PMOS_GSD"             - Generic P-MOSFET (G, S, D pins)
  "Device:Q_NPN_BCE"              - Generic NPN BJT (B, C, E pins)
  "Device:Q_PNP_BCE"              - Generic PNP BJT (B, C, E pins)

  Specific parts (WARNING: pins may cluster at symbol center!):
  "Transistor_FET:BSS138"           - N-channel MOSFET (G, S, D)
  "Transistor_FET:IRF540N"          - Power N-channel MOSFET
  "Transistor_BJT:2N2222"           - NPN transistor (B, C, E)
  "Transistor_BJT:2N3904"           - NPN transistor (B, C, E)

  NOTE: Specific part symbols may report ALL pins at nearly the same position
        (within 0.1mm). If this happens, wire to the symbol CENTER instead.

Power Symbols (use sch.add_power_symbol with JUST the name, no library prefix):
  "GND"               - Ground
  "VCC"               - VCC power rail
  "+3V3"              - 3.3V rail
  "+5V"               - 5V rail
  "+12V"              - 12V rail
  "VDD"               - Digital supply
  "VSS"               - Digital ground

Connectors:

# ============================================================
# PHASE 1: PLANNING - Query settings and plan layout
# ============================================================

# First, get the usable area and grid
area = sch.get_usable_area()
grid = sch.get_grid_settings()
print(f"Usable area: X={area['min_x_mm']:.0f}-{area['max_x_mm']:.0f}mm, Y={area['min_y_mm']:.0f}-{area['max_y_mm']:.0f}mm")
print(f"Center: ({area['center_x_mm']:.0f}, {area['center_y_mm']:.0f})")
print(f"Grid: {grid['size_mils']} mils = {grid['size_mm']}mm")

# *** CRITICAL: Start at CENTER and spread outward! ***
center_x = area['center_x_mm']  # ~140mm for A4
center_y = area['center_y_mm']  # ~100mm for A4

# Plan layout CENTERED on the page:
# VCC above center, resistors at center, GND below center
# Use 25mm vertical spacing from center

# Snap positions to grid - all relative to CENTER
vcc_x, vcc_y = sch.snap_to_grid(center_x, center_y - 40)   # Above center
r1_x, r1_y = sch.snap_to_grid(center_x, center_y - 15)     # Upper-center
r2_x, r2_y = sch.snap_to_grid(center_x, center_y + 15)     # Lower-center
gnd_x, gnd_y = sch.snap_to_grid(center_x, center_y + 40)   # Below center

print(f"Layout centered at ({center_x:.0f}, {center_y:.0f}):")

# ============================================================
# PHASE 2: PLACEMENT - Place all symbols on grid
# ============================================================
vcc = sch.add_power_symbol("VCC", Vector2.from_xy_mm(vcc_x, vcc_y))
r1 = sch.add_symbol("Device:R", Vector2.from_xy_mm(r1_x, r1_y))
r2 = sch.add_symbol("Device:R", Vector2.from_xy_mm(r2_x, r2_y))
gnd = sch.add_power_symbol("GND", Vector2.from_xy_mm(gnd_x, gnd_y))
print("Phase 2 complete: All symbols placed")

# ============================================================
# PHASE 3: PIN DISCOVERY - Print pin info
# ============================================================
print("")
print("Pin positions:")
for name, sym in [("VCC", vcc), ("R1", r1), ("R2", r2), ("GND", gnd)]:
    for pin in sym.pins:
        print(f"  {name}.{pin.number}: ({pin.position.x/1e6:.2f}, {pin.position.y/1e6:.2f})mm")

# ============================================================
# PHASE 4: WIRING - Use pin-based wiring (NO coordinate errors!)
# ============================================================

# Wire VCC to R1 pin 1
sch.wire_pins(vcc, "1", r1, "1")

# Wire R1 pin 2 to R2 pin 1
sch.wire_pins(r1, "2", r2, "1")

# Wire R2 pin 2 to GND
sch.wire_pins(r2, "2", gnd, "1")

# Add junction at voltage divider output (R1-R2 connection)
junction_pos = sch.get_pin_position(r1, "2")
sch.add_junction(junction_pos)

# Add output wire from junction
output_x, output_y = sch.snap_to_grid(center_x + 30, junction_pos.y / 1e6)
sch.wire_from_pin(r1, "2", (output_x, output_y))

# Verify no unconnected pins
unconnected = sch.get_unconnected_pins()
if unconnected:
    print("")
    print("WARNING: Unconnected pins found!")
    for pin in unconnected:
        print(f"  {pin['net_name']}")
else:
    print("")
    print("All pins connected successfully!")

# ============================================================
# PHASE 5: FINISHING - Labels and annotation
# ============================================================
sch.add_label("VOUT", Vector2.from_xy_mm(output_x + 2, output_y - 3))
sch.annotate(scope="all", start_number=1)
print("")
print("Schematic complete!")

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Components/wires placed way outside page (e.g., coordinates like 175000)
CAUSE: Using wrong units - likely confusing mils with mm or using raw numbers
FIX:
  1. ALWAYS use Vector2.from_xy_mm() with MILLIMETER values
  2. Valid X range: 10-280mm, valid Y range: 10-185mm (for A4)
  3. Check: if coordinate > 500, it's probably WRONG
  4. First call area = sch.get_usable_area() to see correct ranges
  5. Center is around (136, 97)mm - NOT (136000, 97000)!

PROBLEM: "unconnected-(XX-PadN)" even after using wire_pins()
CAUSE: Pin positions from API may not match KiCad's internal connection points
       (especially for transistors, ICs, and multi-pin components)
FIX:
  1. For simple 2-pin components: wire_pins() should work
  2. For transistors/ICs: Wire to symbol CENTER position instead
  3. Verify visually in KiCad - connection may actually be OK
  4. Use generic symbols (Device:Q_NMOS_GSD) instead of specific parts

PROBLEM: Schematic is too large / outside drawing area
CAUSE: Not centering design in usable area, components too spread out
FIX:
  1. ALWAYS start at CENTER: area['center_x_mm'], area['center_y_mm']
  2. For A4: Start at (~136, ~97), spread outward from there
  3. Use 30-40mm spacing (not 20-25mm)
  4. Keep design within 100mm x 80mm total footprint

PROBLEM: All transistor pins report nearly same position
CAUSE: Known API limitation - some symbols report pins at symbol center
FIX:
  1. Print pin positions to verify clustering
  2. If clustered, wire to symbol.position (the center)
  3. Use simpler circuit topologies
  4. Try generic symbols instead of specific part numbers

PROBLEM: Symbol rotation doesn't change pin positions
CAUSE: Known API limitation - rotation may not transform pins
FIX: Always query pin positions after placement using sch.get_pin_position()

PROBLEM: Components placed outside visible area
CAUSE: Starting placement at (60, 60) or using wrong units
FIX: Use sch.get_usable_area() and start at center_x_mm, center_y_mm

PROBLEM: Components not aligned
CAUSE: Positions not on grid
FIX: Use sch.snap_to_grid(x, y) before creating Vector2

PROBLEM: global labels cause API error
CAUSE: Global label creation may not be fully supported
FIX: Use label_type="local" instead

PROBLEM: sch.save() or sch.refresh() fails
CAUSE: These commands may not be implemented in the API
FIX: User must save manually in KiCad (Ctrl+S)

PROBLEM: Grid settings show "is_default: True"
CAUSE: KiCad IPC API does not expose actual grid settings
FIX: This is expected - the API returns default values (50 mils = 1.27mm)
     If you've changed grid in KiCad, manually adjust snap_to_grid() param
