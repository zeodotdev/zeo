# Python API for controlling KiCad schematics via IPC
# Modular API Structure - sch.symbols.add(), sch.wiring.wire_pins(), etc.
# Updated: 2026-01-21 - Added view, document_ops, export, simulation, design_blocks

################################################################################
#                                                                              #
#                    MANDATORY WORKFLOW - READ FIRST!                          #
#                                                                              #
################################################################################

ALWAYS follow this 5-phase workflow when creating schematics. Do NOT skip phases!

PHASE 1: PLANNING (Before ANY tool calls!)
  - Query page settings and usable area: sch.page.get_usable_area()
  - Query grid settings: sch.page.get_grid_settings()
  - Note the CENTER position (typically 140, 100 for A4)
  - List all required components with their library names
  - Draw ASCII topology showing how components connect
  - Plan positions CENTERED in usable area, aligned to grid
  - Keep total design within ~100mm x 80mm footprint

PHASE 2: PLACEMENT
  - *** START AT CENTER: area['center_x_mm'], area['center_y_mm'] ***
  - Use sch.page.snap_to_grid(x, y) to align positions to grid
  - Place ALL symbols first using grid-aligned positions
  - Use 30-40mm spacing between components (not 20-25mm)
  - Do NOT add any wires yet
  - Verify all components placed successfully

PHASE 3: PIN DISCOVERY
  - Query pin positions using sch.wiring.get_pin_position(symbol, pin_id)
  - Print pin positions to verify they're correct
  - Note: Pin positions are EXACT - use them without modification

PHASE 4: WIRING (Use Pin-Based Wiring!)
  - ALWAYS use pin-based wiring functions:
    * sch.wiring.wire_pins(sym1, pin1, sym2, pin2) - Direct pin-to-pin
    * sch.wiring.wire_path((sym1, pin1), waypoints, (sym2, pin2)) - Routed path
    * sch.wiring.wire_from_pin(sym, pin, (x, y)) - Pin to coordinate
    * sch.wiring.wire_to_pin((x, y), sym, pin) - Coordinate to pin
    * sch.wiring.auto_wire(sym1, pin1, sym2, pin2) - L-shaped routing
  - Add junctions where 3+ wires meet
  - Check for unconnected pins: sch.connectivity.get_unconnected_pins()

PHASE 5: FINISHING
  - Add labels and annotations
  - Run sch.erc.annotate() to assign reference designators
  - Verify no unconnected pins: sch.connectivity.get_unconnected_pins()

================================================================================
HOW TO USE THE SHELL
================================================================================

Use the run_shell tool with mode="sch" to execute Python code for schematic control.

The shell has the following PRE-IMPORTED and READY TO USE:
  - kipy             (the kipy library)
  - sch              (Schematic object - already connected to KiCad!)
  - Vector2          (from kipy.geometry)

IMPORTANT: You do NOT need to import anything - just use `sch` directly!

================================================================================
API STRUCTURE - MODULAR ORGANIZATION
================================================================================

The Schematic class provides access to all functionality through organized
sub-modules:

  sch.crud          - Create, Read, Update, Delete operations
  sch.symbols       - Symbol placement, search, move, copy, rotate, duplicate, replace
  sch.wiring        - Wire creation, pin-based wiring, auto-routing, junctions, no-connects
  sch.labels        - Net labels, power symbols, net naming, net management
  sch.sheets        - Hierarchical sheet management
  sch.erc           - Electrical rules checking and validation
  sch.graphics      - Text, shapes, annotations, groups
  sch.page          - Page settings, title block, grid operations
  sch.library       - Symbol library management and search
  sch.selection     - Editor selection operations
  sch.connectivity  - Net and bus queries
  sch.transform     - Batch transform operations (move, rotate, mirror, align, distribute)
  sch.buses         - Bus definition, routing, and analysis
  sch.view          - Viewport, zoom, highlighting, cross-probe, undo/redo
  sch.simulation    - SPICE simulation settings and execution
  sch.export        - Export operations (netlist, BOM, plot, serialization)
  sch.design_blocks - Design block management and placement
  sch.document_ops  - Document management (create, open, close, save)

================================================================================
CRITICAL CODE FORMATTING RULES
================================================================================

1. DO NOT use backslash-n (\n) inside string literals - it causes SyntaxError!
   BAD:  print(f"\nHello")
   GOOD: print("")
         print("Hello")

2. Use 4 spaces for indentation (Python standard)

3. Keep code blocks SHORT - complex operations may timeout

================================================================================
SCHEMATIC PAGE AND COORDINATE SYSTEM - CRITICAL!
================================================================================

*** COORDINATE SYSTEM (MUST UNDERSTAND!) ***
  - Page origin (0, 0) is at the TOP-LEFT corner of the page
  - X increases to the RIGHT (towards right edge)
  - Y increases DOWNWARD (towards bottom edge)
  - ALL COORDINATES are in MILLIMETERS when using Vector2.from_xy_mm()
  - Internal API uses nanometers (1 mm = 1,000,000 nm)

ALWAYS START by querying the usable area and grid:

  # Get usable drawing area (accounts for title block margins)
  area = sch.page.get_usable_area()
  print(f"Usable area: X={area['min_x_mm']:.0f}-{area['max_x_mm']:.0f}mm")
  print(f"            Y={area['min_y_mm']:.0f}-{area['max_y_mm']:.0f}mm")
  print(f"Center: ({area['center_x_mm']:.0f}, {area['center_y_mm']:.0f})")

  # Get grid settings (NOTE: returns defaults - no API to query actual grid!)
  grid = sch.page.get_grid_settings()
  print(f"Grid: {grid['size_mils']} mils = {grid['size_mm']}mm")
  if grid['is_default']:
      print("(Using default grid - actual grid in KiCad may differ)")

TYPICAL VALUES (A4 landscape, 297x210mm page):
  - Usable X: 20mm to 260mm (left margin to title block)
  - Usable Y: 20mm to 180mm (top margin to title block)
  - Center: (140, 100) approximately
  - Grid: 50 mils = 1.27mm (DEFAULT VALUE)

*** CRITICAL: CENTER YOUR DESIGN! ***
  - ALWAYS start placement at the CENTER of the usable area
  - For A4: Start at (~140, ~100), NOT at (60, 60)!
  - Spread components outward from center
  - Use LARGE spacing: 30-50mm between components (not 20-25mm)

ALWAYS snap positions to grid:
  x, y = sch.page.snap_to_grid(100.5, 75.3)  # Returns grid-aligned coords
  pos = Vector2.from_xy_mm(x, y)  # Creates position in MILLIMETERS

WIRING OPERATIONS (sch.wiring) - ALWAYS USE THESE!
================================================================================

*** CRITICAL: ALWAYS USE PIN-BASED WIRING! ***

DO NOT use add_wire() with manual coordinates - pins won't connect!
ALWAYS use wire_pins(), auto_wire(), or wire_from_pin() instead.

Example of WRONG approach (pins won't connect):
  # WRONG - manual coordinates don't hit exact pin positions!
  sch.wiring.add_wire(Vector2.from_xy_mm(100, 80), Vector2.from_xy_mm(100, 110))

Example of CORRECT approach (pins always connect):
  # CORRECT - uses exact pin positions from KiCad!
  sch.wiring.wire_pins(r1, "1", r2, "2")

================================================================================
IMPORTANT: WHY USE PIN-BASED WIRING?
================================================================================

PROBLEM: Manual coordinate wiring is error-prone:
  # This might NOT connect due to tiny position errors!
  sch.wiring.add_wire(Vector2.from_xy_mm(100, 80), Vector2.from_xy_mm(100, 110))

Even 0.01mm offset = "unconnected" error!

SOLUTION: Pin-based wiring uses EXACT positions:
  # This ALWAYS connects correctly!
  sch.wiring.wire_pins(r1, "2", r2, "1")

ALWAYS prefer (in order of preference):
  1. sch.wiring.auto_wire() - L-shaped routing, works for any pin positions
  2. sch.wiring.connect_to_junction() - for star connections (3+ components)
  3. sch.wiring.wire_pins() - direct connections when pins are aligned
  4. sch.wiring.auto_wire_to_point() - pin to coordinate with L-routing
  5. sch.wiring.wire_from_pin() / wire_to_pin() - for manual junction wiring

# ============================================================
# PHASE 1: PLANNING - Query settings and plan layout
# ============================================================

# First, get the usable area and grid
area = sch.page.get_usable_area()
grid = sch.page.get_grid_settings()
print(f"Usable area: X={area['min_x_mm']:.0f}-{area['max_x_mm']:.0f}mm, Y={area['min_y_mm']:.0f}-{area['max_y_mm']:.0f}mm")
print(f"Center: ({area['center_x_mm']:.0f}, {area['center_y_mm']:.0f})")
print(f"Grid: {grid['size_mils']} mils = {grid['size_mm']}mm")

# *** CRITICAL: Start at CENTER and spread outward! ***
center_x = area['center_x_mm']  # ~140mm for A4
center_y = area['center_y_mm']  # ~100mm for A4

# Plan layout CENTERED on the page
# Snap positions to grid - all relative to CENTER
vcc_x, vcc_y = sch.page.snap_to_grid(center_x, center_y - 40)
r1_x, r1_y = sch.page.snap_to_grid(center_x, center_y - 15)
r2_x, r2_y = sch.page.snap_to_grid(center_x, center_y + 15)
gnd_x, gnd_y = sch.page.snap_to_grid(center_x, center_y + 40)

# ============================================================
# PHASE 2: PLACEMENT - Place all symbols on grid
# ============================================================
vcc = sch.labels.add_power("VCC", Vector2.from_xy_mm(vcc_x, vcc_y))
r1 = sch.symbols.add("Device:R", Vector2.from_xy_mm(r1_x, r1_y))
r2 = sch.symbols.add("Device:R", Vector2.from_xy_mm(r2_x, r2_y))
gnd = sch.labels.add_power("GND", Vector2.from_xy_mm(gnd_x, gnd_y))
print("Phase 2 complete: All symbols placed")

# ============================================================
# PHASE 3: PIN DISCOVERY - Print pin info
# ============================================================
print("")
print("Pin positions:")
for name, sym in [("VCC", vcc), ("R1", r1), ("R2", r2), ("GND", gnd)]:
    for pin in sym.pins:
        print(f"  {name}.{pin.number}: ({pin.position.x/1e6:.2f}, {pin.position.y/1e6:.2f})mm")

# ============================================================
# PHASE 4: WIRING - Use pin-based wiring (NO coordinate errors!)
# ============================================================

# Wire VCC to R1 pin 1
sch.wiring.wire_pins(vcc, "1", r1, "1")

# Wire R1 pin 2 to R2 pin 1
sch.wiring.wire_pins(r1, "2", r2, "1")

# Wire R2 pin 2 to GND
sch.wiring.wire_pins(r2, "2", gnd, "1")

# Add junction at voltage divider output (R1-R2 connection)
junction_pos = sch.wiring.get_pin_position(r1, "2")
sch.wiring.add_junction(junction_pos)

# Add output wire from junction
output_x, output_y = sch.page.snap_to_grid(center_x + 30, junction_pos.y / 1e6)
sch.wiring.wire_from_pin(r1, "2", (output_x, output_y))

# Verify no unconnected pins
unconnected = sch.connectivity.get_unconnected_pins()
if unconnected:
    print("")
    print("WARNING: Unconnected pins found!")
    for pin in unconnected:
        print(f"  {pin['net_name']}")
else:
    print("")
    print("All pins connected successfully!")

# ============================================================
# PHASE 5: FINISHING - Labels and annotation
# ============================================================
sch.labels.add_local("VOUT", Vector2.from_xy_mm(output_x + 2, output_y - 3))
sch.erc.annotate(scope="all", start_number=1)
print("")
print("Schematic complete!")

================================================================================
IMPORTANT UNITS - READ CAREFULLY!
================================================================================

*** COORDINATE VALUES - AVOID CONFUSION! ***

When placing components or creating wires, ALWAYS use MILLIMETERS:
  - Standard A4 page is ~297mm x 210mm
  - Typical center is around (140, 100) mm
  - Component positions should be in range 10-280mm for X, 10-185mm for Y

WRONG (values too large - these would be meters!):
  pos = Vector2.from_xy_mm(175000, 195000)  # 175 METERS - WRONG!

CORRECT (reasonable millimeter values):
  pos = Vector2.from_xy_mm(140, 100)  # 140mm, 100mm - CORRECT!

KiCad uses NANOMETERS internally:
  - 1 mm = 1,000,000 nm (one million nanometers)
  - Use Vector2.from_xy_mm(x_mm, y_mm) for millimeter input (RECOMMENDED!)
  - Use Vector2.from_xy_mils(x_mils, y_mils) for mils input
  - Use Vector2.from_xy(x_nm, y_nm) only if you need nanometer precision

Converting from API positions to mm for debugging:
  x_mm = pin.position.x / 1e6  # Divide nm by 1,000,000 to get mm
  y_mm = pin.position.y / 1e6

*** SANITY CHECK FOR COORDINATES ***
  - If X or Y > 500, you're probably using wrong units!
  - A4 page is only 297mm wide - values like 175000 are WRONG
  - Always check: area = sch.page.get_usable_area() to see valid ranges

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Wires don't connect to pins (ERC shows "unconnected" errors)
CAUSE: Using add_wire() with manual coordinates instead of pin-based wiring
FIX:
  1. NEVER use add_wire() with manual coordinates for component connections
  2. ALWAYS use wire_pins(sym1, "pin1", sym2, "pin2")
  3. Or use auto_wire() for L-shaped routing
  4. Only use add_wire() for connecting existing wires to junctions

PROBLEM: sch.library.search_symbols() times out
CAUSE: This function parses ALL library files client-side - extremely slow
FIX:
  1. Use known lib_ids directly (see COMMON LIBRARY SYMBOLS section)
  2. Example: sch.symbols.add("Device:R", pos) instead of searching
  3. For MOSFETs, use "Device:Q_NMOS_GSD" or "Transistor_FET:BSS138"

PROBLEM: Components/wires placed way outside page (e.g., coordinates like 175000)
CAUSE: Using wrong units - likely confusing mils with mm or using raw numbers
FIX:
  1. ALWAYS use Vector2.from_xy_mm() with MILLIMETER values
  2. Valid X range: 10-280mm, valid Y range: 10-185mm (for A4)
  3. Check: if coordinate > 500, it's probably WRONG
  4. First call area = sch.page.get_usable_area() to see correct ranges
  5. Center is around (140, 100)mm - NOT (140000, 100000)!

PROBLEM: "unconnected-(XX-PadN)" even after using wire_pins()
CAUSE: Pin positions from API may not match KiCad's internal connection points
FIX:
  1. For simple 2-pin components: wire_pins() should work
  2. For transistors/ICs: Use auto_wire() which handles L-routing
  3. Verify visually in KiCad - connection may actually be OK
  4. Use generic symbols (Device:Q_NMOS_GSD) instead of specific parts

PROBLEM: Schematic is too large / outside drawing area
CAUSE: Not centering design in usable area
FIX:
  1. ALWAYS start at CENTER: area['center_x_mm'], area['center_y_mm']
  2. For A4: Start at (~140, ~100), spread outward
  3. Use 30-40mm spacing (not 20-25mm)
  4. Keep design within 100mm x 80mm total footprint

PROBLEM: All transistor pins report nearly same position
CAUSE: Known API limitation - some symbols report pins at symbol center
FIX:
  1. Print pin positions to verify clustering
  2. Use auto_wire() which creates proper L-routing
  3. Try generic symbols (Device:Q_NMOS_GSD)

PROBLEM: Symbol rotation doesn't change pin positions
CAUSE: Known API limitation - rotation may not transform pins
FIX: Always query pin positions after placement

PROBLEM: Global labels cause API error
CAUSE: Global label creation may not be fully supported
FIX: Use label_type="local" instead

PROBLEM: sch.save() fails
CAUSE: Save may not be implemented or may require file permissions
FIX: User must save manually in KiCad (Ctrl+S)
